# v2 온보딩 예시 안내 개선 설계

작성일: 2026-04-27

## 배경

Truck Harvester v2의 주요 사용자는 인터넷 사용에 익숙하지 않은 40대 중고차 딜러와 막 직장 생활을 시작한 비개발자 직원이다. 현재 온보딩은 각 영역을 스포트라이트로 강조하고 설명하는 구조라 방향은 좋지만, 사용자가 실제로 어떤 행동을 해야 하는지와 그 행동 뒤에 어떤 화면이 나타나는지까지는 충분히 보여주지 못한다.

특히 매물 주소 입력 단계에서는 사용자가 브라우저 주소창의 전체 주소를 붙여넣어야 한다는 점을 놓칠 수 있다. 드물게 `https://www.truck-no1.co.kr/model/` 앞부분을 빼고 `DetailView.asp?...`부터 붙여넣어 안 된다고 느끼는 사례가 있다. 저장 폴더 단계에서는 선택한 폴더 안에 차량별 폴더가 어떻게 생기는지 예측하기 어렵고, 진행 상황 단계에서는 저장 중과 저장 완료 상태가 어떻게 구분되는지 미리 알기 어렵다.

이 설계는 기존 `/` route와 legacy 동작을 유지하면서 `/v2` 범위에서만 적용한다. v2에는 Sentry와 watermark 기능을 추가하지 않는다.

## 목표

- 온보딩을 길게 만들지 않고 3단계 안에서 사용자가 해야 할 행동을 분명히 보여준다.
- 설명 문구만이 아니라 작은 예시 카드로 입력 전후와 저장 결과를 보여준다.
- 브라우저 주소창의 전체 매물 주소를 복사해야 한다는 점을 쉬운 말로 안내한다.
- 복사한 내용 안에 매물 주소가 들어 있으면 자동으로 찾는다는 점을 알려준다.
- 저장 폴더 선택 뒤의 파일 정리 결과를 예시로 보여준다.
- 저장 중과 저장 완료 상태, 선택형 완료 알림을 한 번에 이해할 수 있게 한다.
- 기존 스포트라이트, 이전/다음, 닫기 흐름은 유지하고 키보드 방향키 이동을 추가한다.

## 확정 접근

접근은 `짧은 예시 카드형`으로 확정한다.

기존 온보딩 단계 수는 3단계로 유지한다. 각 단계는 현재처럼 관련 화면 요소를 밝게 남기고 나머지 화면을 어둡게 처리한다. 설명 카드는 해당 요소 가까이에 붙어서 나타나며, 설명 카드 안에 단계별 작은 예시 블록을 추가한다.

이 방식은 별도의 따라 하기 튜토리얼을 만들지 않는다. 사용자는 실제 입력이나 폴더 선택을 강제로 하지 않아도, 각 단계에서 "무엇을 하면 어떤 결과가 나오는지"를 짧게 보고 이해한다.

## 사용자 흐름

1. 사용자가 `도움말`을 누르거나 첫 방문 조건에 따라 온보딩이 열린다.
2. 1단계에서 `매물 주소 넣기` 영역이 강조된다.
3. 설명 카드는 주소창의 전체 매물 주소를 붙여넣는 예시와 확인 완료 칩 예시를 보여준다.
4. 사용자가 `다음` 또는 `→` 방향키를 누르면 저장 폴더 단계로 이동한다.
5. 2단계에서 `저장 폴더 선택` 영역이 강조된다.
6. 설명 카드는 저장 폴더를 고른 뒤 차량별 폴더가 생기는 예시를 보여준다.
7. 사용자가 `다음` 또는 `→` 방향키를 누르면 진행 상황 단계로 이동한다.
8. 3단계에서 `저장 진행 상황` 영역이 강조된다.
9. 설명 카드는 저장 중과 저장 완료 예시를 나란히 보여주고 완료 알림을 짧게 안내한다.
10. 사용자가 `마치기`, 마지막 단계의 `→` 방향키, 또는 `Esc`를 누르면 온보딩이 닫힌다.

## 단계별 문구와 예시

### 1단계: 매물 주소 넣기

- 제목: `매물 주소를 넣어요`
- 설명: `주소창에 있는 매물 주소를 처음부터 끝까지 복사해 붙여넣으세요. 복사한 내용 안에 매물 주소가 들어 있으면 자동으로 찾아요.`
- 좋은 예시:
  - `https://www.truck-no1.co.kr/model/DetailView.asp?...`
- 주의 예시:
  - `DetailView.asp?...처럼 앞부분이 빠진 주소는 찾지 못할 수 있어요.`
- 결과 예시:
  - `덤프 메가트럭 4.5톤`
  - `확인 완료`

### 2단계: 저장할 곳 고르기

- 제목: `저장할 곳을 고르세요`
- 설명: `사진과 차량 정보가 선택한 폴더 안에 차량별로 정리됩니다.`
- 동작 예시:
  - `저장 폴더 고르기`
  - `truck-test`
- 결과 예시:

```text
truck-test
└─ 서울80바1234
   ├─ 사진 1
   ├─ 사진 2
   └─ 차량정보.txt
```

### 3단계: 저장 진행 확인하기

- 제목: `저장되는지 확인해요`
- 설명: `저장 중인 매물과 저장이 끝난 매물을 여기서 확인할 수 있어요.`
- 상태 예시:
  - `저장 중`
  - `저장 완료`
- 알림 안내:
  - `작업이 끝났을 때 알려주는 완료 알림도 원하면 켤 수 있어요.`

## 구조 설계

기존 `TourStep` 데이터에 예시 카드 정보를 추가한다. 온보딩 단계 정의는 제목, 설명, 대상 요소, fallback 요소, 예시 타입을 포함한다.

예시 카드 렌더링은 `TourOverlay` 안에 모든 마크업을 직접 쌓지 않고, 단계의 예시 타입을 받아 작게 렌더링하는 별도 UI로 분리한다. 예시 타입은 다음처럼 제한한다.

- `url-example`: 전체 매물 주소 입력과 확인 완료 칩 예시
- `folder-example`: 저장 폴더 선택과 차량별 폴더 미리보기
- `progress-example`: 저장 중, 저장 완료, 완료 알림 예시

온보딩 상태는 현재 단계, 이전, 다음, 닫기만 관리한다. 예시 카드는 실제 URL 입력, 폴더 선택, 저장 진행 상태를 변경하지 않는 순수한 안내 UI다.

## 화면 동작

- 온보딩이 열리면 현재 단계의 `data-tour` 대상 요소만 밝게 남긴다.
- 나머지 화면은 어둡게 처리하되, 설명 카드와 강조 박스의 대비는 충분히 유지한다.
- 설명 카드는 가능한 한 강조 요소 가까이에 배치한다.
- 작은 화면에서는 설명 카드가 강조 요소를 가리지 않도록 위쪽 또는 아래쪽으로 자동 배치한다.
- 설명 카드가 길어질 수 있으므로 최대 높이를 두고 내부 스크롤을 허용한다.
- 실제 작업을 강제하지 않는다. 온보딩을 보면서 입력, 저장 폴더 선택, 저장 시작을 요구하지 않는다.

## 키보드와 접근성

- `이전` 버튼은 이전 단계로 이동한다.
- `다음` 버튼은 다음 단계로 이동한다.
- 마지막 단계의 기본 버튼은 `마치기`로 표시하고 온보딩을 닫는다.
- `←` 방향키는 이전 단계로 이동한다.
- `→` 방향키는 다음 단계로 이동한다.
- 첫 단계에서 `←` 방향키를 누르면 아무 일도 일어나지 않는다.
- 마지막 단계에서 `→` 방향키를 누르면 `마치기`와 같은 동작으로 온보딩을 닫는다.
- `Esc`는 온보딩을 닫는다.
- 입력란, textarea, select, contenteditable 요소에 포커스가 있을 때는 방향키가 온보딩 이동을 트리거하지 않는다.
- 현재 단계와 전체 단계 수는 화면에 표시하고 스크린 리더가 읽을 수 있게 한다.
- 상태 예시는 색만으로 구분하지 않고 텍스트와 아이콘을 함께 사용한다.
- 온보딩 카드 안의 버튼들은 논리적인 탭 순서를 유지한다.

## 애니메이션

- 온보딩은 첫 사용 또는 도움말에서만 보이는 흐름이므로 짧은 애니메이션을 허용한다.
- 단계 전환 시 강조 박스와 어두운 영역은 부드럽게 위치가 바뀐다.
- 설명 카드는 200ms 안팎으로 살짝 떠오르듯 나타난다.
- 반복 업무 도구이므로 튀는 효과, 긴 지연, 장식적인 움직임은 사용하지 않는다.
- `prefers-reduced-motion` 또는 기존 `/v2` reduced-motion 프리셋을 존중한다.
- 애니메이션은 opacity, transform, position 변경 중심으로 제한한다.

## 테스트 계획

- 온보딩 단계 데이터가 3단계를 유지하는지 테스트한다.
- 각 단계가 올바른 예시 타입을 가지는지 테스트한다.
- 주소 단계에서 전체 매물 주소 예시와 앞부분이 빠진 주소 주의 문구가 표시되는지 테스트한다.
- 저장 폴더 단계에서 차량별 폴더 미리보기가 표시되는지 테스트한다.
- 진행 상황 단계에서 저장 중, 저장 완료, 완료 알림 안내가 표시되는지 테스트한다.
- `이전`, `다음`, `마치기`, `Esc` 동작을 테스트한다.
- `←`, `→` 방향키 동작을 테스트한다.
- 입력란에 포커스가 있을 때 방향키가 온보딩 이동을 트리거하지 않는지 테스트한다.
- 작은 화면에서 카드가 화면 밖으로 벗어나지 않는지 확인한다.
- reduced-motion 환경에서 애니메이션이 줄어드는지 확인한다.

## 범위 밖

- 온보딩 단계 수를 4단계 이상으로 늘리지 않는다.
- 실제 따라 하기 모드나 샘플 데이터를 주입하는 기능은 만들지 않는다.
- 저장 폴더 기억 기능은 다시 도입하지 않는다.
- 진행 중 개별 매물 취소 기능은 이번 범위에서 다루지 않는다.
- legacy `/` route, `src/shared`, `src/widgets`, `src/app/page.tsx` 동작 변경은 제외한다.
- v2 watermark 기능 추가는 제외한다.
- v2 Sentry 추가는 제외한다.
